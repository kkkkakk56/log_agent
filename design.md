# Design System: 心记 / 做记 (journal-agent)

这是 journal-agent 项目的设计系统文档,供 AI 编码 Agent 在生成 UI 代码时读取,确保视觉和交互风格一致。

项目定位:一个先本地可用、用多个独立 Park 组织日记、知识库、项目实验、计划的个人信息 App。气质对标 Apple Journal / Notes / Reminders 的原生 iOS 风格——**克制、柔和、留白充足、触感温暖**,不是 apple.com 的营销页风格。

---

## 1. 视觉哲学与气质

这个 App 的视觉核心可以用一句话概括:**像一本柔软的日记本,而不是一个生产力工具**。

用户打开 App 应该感到平静,而不是被信息密度压迫。每一处交互都应该有轻微的物理反馈感——点击有弹性、出现有淡入、完成有仪式。**所有视觉元素都退到内容之后**,让文字本身成为主角。

这不是极简主义的美学偏好,而是对"个人沉淀"这件事的尊重——用户在这里写下的是自己的情绪、思考、项目进展,界面不应该抢走这些内容的注意力。

### 核心气质特征

- 背景使用系统级浅色(`systemGroupedBackground` / `systemBackground`),**避免纯白**
- 卡片是白色的容器,通过极轻的阴影和圆角从背景中"浮"出
- 字体优先 iOS 系统栈(`-apple-system, SF Pro, PingFang SC`),字重克制,主要用 400 和 600
- Accent 色**只有一种**(琥珀金),用于重要标记、当前选中、关键操作
- 动效是"慢半拍"的——淡入、滑入、轻微缩放,没有爆炸式的转场
- 列表和卡片之间用**颜色分层**和**间距**区分,几乎不用分割线
- 磨砂玻璃效果用于浮层、抽屉、Agent 面板——制造"浮在内容之上"的层次感

### 关键避讳

- ❌ 不用 apple.com 营销页的黑白交替 cinematic 布局
- ❌ 不用 980px 胶囊 CTA(那是网页设计)
- ❌ 不用饱和度高的纯红/纯绿——所有颜色要带灰度或偏暖
- ❌ 不用硬边框、实线分隔——用背景色差和间距代替
- ❌ 不用纯黑 `#000000` 做深色模式背景——用 iOS 的系统深色
- ❌ 不用表情符号做 UI 图标(emoji 在不同 iOS 版本渲染差异大)——用 SF Symbols

## 2. 色彩系统

### 基础规则

- **背景分三层**:分组背景 < 标准背景 < 卡片背景,通过明度递增制造层次
- **文本分三层**:主要 > 次要 > 三级,通过灰度递减制造层次
- **Accent 只用一种琥珀金**,不引入其他彩色作 UI(类型颜色除外,见 §2.4)
- 所有颜色都要同时定义浅色和深色模式

### 2.1 背景色

**浅色模式**
- `bg-grouped`: `#F2F2F7` — 列表分组背景(iOS systemGroupedBackground)
- `bg-base`: `#FFFFFF` — 标准页面背景
- `bg-card`: `#FFFFFF` — 卡片背景(和 bg-base 同色,通过阴影区分)
- `bg-elevated`: `#FFFFFF` — 浮层/抽屉背景

**深色模式**
- `bg-grouped`: `#000000` — 列表分组背景(iOS 深色模式标准)
- `bg-base`: `#1C1C1E` — 标准页面背景
- `bg-card`: `#2C2C2E` — 卡片背景
- `bg-elevated`: `#3A3A3C` — 浮层/抽屉背景

### 2.2 文本色

**浅色模式**
- `text-primary`: `#1D1D1F` — 主要文本(不用纯黑,iOS 标准稍暖)
- `text-secondary`: `rgba(60, 60, 67, 0.6)` — 次要文本(标签、描述)
- `text-tertiary`: `rgba(60, 60, 67, 0.3)` — 三级文本(占位、禁用)

**深色模式**
- `text-primary`: `#FFFFFF`
- `text-secondary`: `rgba(235, 235, 245, 0.6)`
- `text-tertiary`: `rgba(235, 235, 245, 0.3)`

### 2.3 Accent 色

**单一 accent,琥珀金**,语义是"值得关注""珍贵的"。用于重要标记、当前选中项、关键操作按钮。

- `accent`: `#D4912A` — 标准琥珀金
- `accent-soft`: `rgba(212, 145, 42, 0.12)` — 琥珀金填充(用于重要卡片左侧色条的淡背景)
- `accent-strong`: `#B8791F` — 按压/悬停时的稍深琥珀金

**不要用标准 iOS Blue (`#007AFF`) 作为 accent**——那会让 App 看起来和所有 iOS App 都一样,琥珀金是这个 App 的辨识度。

### 2.4 类型色(做记专用)

做记 Park 里,记录按类型用颜色区分。这些颜色**只用于类型标识**,不用于其他 UI 元素。所有颜色都是"带灰度的"柔和版本,不是饱和色。

- `type-progress`: `#5BAA6F` — 进展(柔和绿,不是纯绿)
- `type-review`: `#C75C5C` — 复盘(柔和红,偏砖红)
- `type-decision`: `#5B8FBF` — 决策(柔和蓝)
- `type-issue`: `#D4A24A` — 问题(柔和琥珀,和 accent 有区分)

每个类型色都有一个 `-soft` 变体(透明度 0.12)用于卡片左侧色条。

### 2.5 状态色

- `success`: `#5BAA6F`(复用 type-progress)
- `warning`: `#D4A24A`(复用 type-issue)
- `danger`: `#C75C5C`(复用 type-review)——用于删除、取消类破坏性操作
- `info`: `#5B8FBF`(复用 type-decision)

### 2.6 分隔符

**极少使用分隔线**。如果必须使用:
- `separator`: `rgba(60, 60, 67, 0.12)` — 浅色模式
- `separator-dark`: `rgba(84, 84, 88, 0.35)` — 深色模式

优先用间距和背景色差代替。

## 3. 字体系统

### 3.1 字体栈

```css
font-family: -apple-system, BlinkMacSystemFont, 'SF Pro Text', 'SF Pro Display',
             'PingFang SC', 'Helvetica Neue', Helvetica, Arial, sans-serif;
```

iOS 会自动根据字号选择 SF Pro Display(20px+)或 SF Pro Text(≤19px)。中文字体优先苹方。

### 3.2 字号层级

所有字号都采用**负字符间距**,这是 Apple 字体哲学的核心——即便是正文也要 tracking tight。

| 角色 | 字号 | 字重 | 行高 | 字间距 | 使用场景 |
|---|---|---|---|---|---|
| Large Title | 34px | 700 | 1.2 | -0.4px | 页面主标题(如"心记""做记") |
| Title 1 | 28px | 600 | 1.2 | -0.38px | 区块大标题 |
| Title 2 | 22px | 600 | 1.27 | -0.3px | 卡片标题 |
| Title 3 | 20px | 600 | 1.25 | -0.26px | 小区块标题 |
| Headline | 17px | 600 | 1.29 | -0.28px | 列表项标题、强调正文 |
| Body | 17px | 400 | 1.47 | -0.374px | 标准正文 |
| Callout | 16px | 400 | 1.31 | -0.26px | 提示性段落 |
| Subhead | 15px | 400 | 1.33 | -0.24px | 次要描述 |
| Footnote | 13px | 400 | 1.38 | -0.08px | 时间戳、元信息 |
| Caption 1 | 12px | 400 | 1.33 | 0 | 辅助说明 |
| Caption 2 | 11px | 400 | 1.18 | 0.06px | 最小级辅助文字 |

### 3.3 字体原则

- **字重克制**:主要用 400(正文)和 600(标题/强调)。不用 300(太细),不用 700+(太重)。
- **负字符间距通用**:不只是标题,正文也要 -0.374px。这是让文字看起来"精密"的关键。
- **正文左对齐**:所有段落文字左对齐,只有居中摆放的小卡片或空状态可以用居中。
- **行高两极化**:标题紧凑(1.2-1.3),正文宽松(1.47),通过行高差异自然产生层级。
- **中英文混排时不要手动加空格**:依赖字体本身的处理,保持简洁。

## 4. 间距系统

### 4.1 基础单位

基础单位 `4px`。所有间距都是 4 的倍数,形成视觉节奏。

| 名称 | 值 | 使用场景 |
|---|---|---|
| xs | 4px | 图标与文字间距、紧凑元素 |
| sm | 8px | 小元素间距、图标内边距 |
| md | 12px | 卡片内次要间距 |
| base | 16px | **最常用**,卡片内边距、列表项内边距 |
| lg | 20px | 区块间距 |
| xl | 24px | 大区块间距 |
| 2xl | 32px | 页面区块的呼吸空间 |
| 3xl | 48px | 超大留白,hero 区域 |

### 4.2 页面级间距

- 页面左右安全边距:**20px**(iPhone 标准)
- 区块之间垂直间距:**24px-32px**
- 卡片列表之间:**8px**(紧凑感)或 **12px**(舒适感),不要 16px+,会显得松散

### 4.3 卡片内部间距

- 卡片内边距:**16px**(上下左右统一)
- 卡片内文字行间(标题到正文):**4px**
- 卡片内图标到文字:**8px**

### 4.4 列表项间距

- 列表项上下内边距:**12px**
- 列表项左右内边距:**16px**
- 列表项之间:不用 margin,用浅背景色和卡片化区分

## 5. 圆角系统

圆角是柔和触感的关键。这个 App 的圆角**整体偏大**,强化"柔软日记本"的触感。

| 名称 | 值 | 使用场景 |
|---|---|---|
| radius-sm | 6px | 小标签、小徽章 |
| radius-base | 10px | 按钮、输入框 |
| radius-card | 14px | 卡片(比 iOS 标准 12px 稍大,更柔) |
| radius-lg | 18px | 浮层、抽屉、大容器 |
| radius-pill | 999px | 胶囊按钮(分段控制器、筛选条) |
| radius-circle | 50% | 头像、圆形按钮(Agent 头像、麦克风) |

**禁止使用 0 圆角的矩形**(除了分隔线和全屏背景)。所有可点击元素、所有卡片都要有圆角。

## 6. 阴影系统

阴影**极度克制**——大多数元素没有阴影,通过背景色差和圆角自然"浮起"。只有真正需要强调层次的元素才用阴影。

### 6.1 阴影层级

| 层级 | 样式 | 使用场景 |
|---|---|---|
| Level 0 | 无阴影 | 大多数列表项、扁平内容 |
| Level 1(卡片) | `0 1px 3px rgba(0, 0, 0, 0.04), 0 1px 2px rgba(0, 0, 0, 0.03)` | 标准卡片,几乎不可见但有触感 |
| Level 2(浮起) | `0 2px 8px rgba(0, 0, 0, 0.06), 0 4px 16px rgba(0, 0, 0, 0.04)` | 置顶卡片、选中状态 |
| Level 3(浮层) | `0 8px 32px rgba(0, 0, 0, 0.08), 0 16px 48px rgba(0, 0, 0, 0.06)` | 抽屉、模态面板、Agent 浮窗 |
| Level 4(最高) | `0 20px 60px rgba(0, 0, 0, 0.12)` | 几乎不用,只在强调性浮窗 |

### 6.2 阴影哲学

- 阴影永远是**柔和的、多层叠加的**——不用单层重阴影
- 阴影是**向下散射**的——y 轴正偏移,模拟上方光源
- 深色模式下阴影更淡(`rgba` 的 alpha 降到 0.02-0.04)——深色模式靠背景色差更多
- **不用 inner shadow**——不符合"柔软纸张"的触感

## 7. 毛玻璃效果

毛玻璃(backdrop-filter blur)是这个 App 的**签名视觉元素**,用于强化"浮在内容之上"的层次。

### 7.1 使用场景

- 导航栏吸顶
- 抽屉(知识库仓库抽屉、类型筛选抽屉等)
- Agent 浮窗聊天面板
- 模态面板
- 底部 tab 栏(如果有)

### 7.2 标准参数

**浅色模式**
```css
background: rgba(255, 255, 255, 0.72);
backdrop-filter: saturate(180%) blur(20px);
-webkit-backdrop-filter: saturate(180%) blur(20px);
```

**深色模式**
```css
background: rgba(28, 28, 30, 0.72);
backdrop-filter: saturate(180%) blur(20px);
-webkit-backdrop-filter: saturate(180%) blur(20px);
```

### 7.3 毛玻璃规则

- **不做不透明导航**——毛玻璃是这个 App 视觉 identity 的一部分,对齐 F013 的 Apple Journal 气质
- 毛玻璃层要有**细微边框**(`border-top: 1px solid rgba(0,0,0,0.04)`)作为明度切分
- iOS WKWebView 完全支持,不需要 fallback
- 毛玻璃内的文字要比普通区域**字重稍重或对比度稍强**,避免被背景噪点干扰

## 8. 核心组件规范

### 8.1 按钮

**Primary(主按钮)**
- 背景:`accent`(琥珀金 `#D4912A`)
- 文字:白色,17px / 600
- 内边距:14px 24px
- 圆角:`radius-base`(10px)
- 按下状态:背景变 `accent-strong`,整体 scale(0.98)
- 最小触摸目标:44px 高度
- 使用场景:确认、保存、创建类关键操作

**Secondary(次要按钮)**
- 背景:`accent-soft`(透明琥珀)
- 文字:`accent`,17px / 600
- 其他同 Primary
- 使用场景:次要确认、替代选项

**Ghost(幽灵按钮)**
- 背景:透明
- 文字:`accent`,17px / 400
- 无边框
- 使用场景:取消、跳过、次要链接

**Destructive(破坏性)**
- 背景:`danger`(柔和红 `#C75C5C`)
- 文字:白色
- 使用场景:删除、清空等不可逆操作,**一定配合二次确认**

**Icon Button(图标按钮)**
- 纯图标,44×44px 触摸区
- SF Symbols 图标,22-24px 大小
- 颜色:`text-secondary`(默认)或 `accent`(激活)
- 按下:0.1s 透明度降到 0.6

### 8.2 输入框

**标准文本输入**
- 背景:`bg-grouped`(浅灰)
- 文字:17px / 400 / `text-primary`
- 内边距:12px 16px
- 圆角:`radius-base`(10px)
- 占位文字:`text-tertiary`
- 聚焦状态:背景变白 + `0 0 0 3px accent-soft` 外发光环
- 无边框(靠背景色差区分)

**多行文本区**
- 同上,但至少 5 行高度
- 行高 1.47(和 Body 一致)
- 自动高度扩展,最大 300px 后内部滚动

**搜索框**
- 圆角 `radius-pill`(999px)
- 左侧带 `magnifyingglass` SF Symbol(14px)
- 右侧带清空按钮(有内容时显示)
- 背景 `bg-grouped`

### 8.3 卡片

**标准笔记卡片**
- 背景:`bg-card`(白)
- 圆角:`radius-card`(14px)
- 阴影:Level 1
- 内边距:16px
- 卡片之间间距:8-12px
- 结构(从上到下):
  - 标题行(Headline 17px / 600)
  - 正文预览(Body 17px / 400,最多 3 行,溢出省略)
  - 元信息行(Footnote 13px / `text-secondary`,时间 + 标签等)

**重要标记的卡片**
- 左侧出现 **3px 宽的琥珀色竖条**(`accent`,从上到下贯穿)
- 卡片右上角有小书签图标(SF Symbol `bookmark.fill`,`accent` 色,14px)
- 其他保持不变——克制的强调,不抢视觉

**置顶的卡片**
- 出现在列表顶部的"📌 置顶"区块内
- 区块背景比普通背景稍微深一点(`bg-grouped` 上叠一层半透明琥珀)
- 卡片本身样式不变
- 区块标题用 Caption 1 / `text-secondary`,左侧带小图钉 SF Symbol(`pin.fill`,11px)

**已完成 Todo 卡片**
- 整体透明度降到 0.5
- 标题加 `text-decoration: line-through`
- 自动折叠到底部"已完成"分组

### 8.4 列表与分组

**分组列表**(类似 iOS Settings)
- 整体背景:`bg-grouped`
- 分组容器:`bg-card` 背景,`radius-card` 圆角
- 分组头:Caption 1 / uppercase / `text-secondary`,左右 20px padding
- 列表项:高度 44px 起步,16px 左右内边距
- 列表项之间:无边距,只有极浅的内部分隔(`separator`,从图标后开始而不是贯穿)

**时间线列表**(心记默认)
- 按日期分组,日期头用 Title 3 / 600
- 日期头下面是当天的卡片列表
- 日期之间用 32px 垂直间距

### 8.5 导航

**顶部导航栏**
- 高度:44px(标准)或 96px(带大标题时)
- 背景:毛玻璃(见 §7.2)
- 大标题模式:左对齐 Large Title(34px / 700),滚动后收缩为标准 17px 标题
- 左右按钮:Icon Button,`accent` 色
- 滚动到顶时无底部边框,滚动后出现 1px 分隔

**抽屉侧栏**(知识库仓库抽屉)
- 宽度:屏幕宽度的 80%,最大 320px
- 背景:毛玻璃
- 阴影:Level 3
- 从左侧滑入,0.3s ease-out 动画
- 遮罩层:`rgba(0, 0, 0, 0.3)`,点击关闭
- 顶部有关闭按钮,底部有"新增仓库"主按钮

### 8.6 分段控制器

用于 Park 内的视图切换(时间线/搜索/日历/地图)。

- 容器:`bg-grouped` 背景,`radius-pill` 圆角
- 选项:文字 Subhead(15px / 600 激活,15px / 400 未激活)
- 选中项:白色背景(`bg-card`)+ Level 1 阴影,`radius-pill`
- 未选中:透明背景,`text-secondary`
- 切换动画:0.25s,选中背景平滑位移(不是瞬间切换)

### 8.7 Agent 浮窗

- 浮窗位置:右下角(可拖拽)
- 触发头像:56×56px 圆形,Level 2 阴影,呼吸光晕动画(静态时)
- 展开面板:
  - 圆角 `radius-lg`(18px)
  - 毛玻璃背景
  - Level 3 阴影
  - 从右下角缩放+淡入展开,0.35s cubic-bezier(0.16, 1, 0.3, 1)
  - 内部结构:顶部对话列表、中间消息流、底部输入框

### 8.8 Toast 与通知

- 出现位置:顶部,距离安全区 16px
- 背景:毛玻璃 + `radius-card`
- 文字:Subhead / 600
- 左侧可选 SF Symbol 图标
- 持续时间:2.5s(成功)/ 3.5s(错误)
- 动画:从顶部下滑 + 淡入,0.3s

## 9. 图标系统

### 9.1 图标来源

**必须使用 SF Symbols**,不用 emoji,不用自定义图标(除非 SF Symbols 找不到)。

SF Symbols 原因:
- 和 iOS 系统气质完美契合
- 支持自动粗细变化跟随字体
- 支持 iOS 原生动画(如 `bounce`、`variableColor`)
- 支持深色模式自动反色
- 免费,苹果官方维护

### 9.2 常用图标对应

| 功能 | SF Symbol |
|---|---|
| 重要标记 | `bookmark.fill` |
| 置顶 | `pin.fill` |
| Todo 未完成 | `circle` |
| Todo 已完成 | `checkmark.circle.fill` |

## 10. 动效系统

对齐 F020 缓冲动效已定的原则:**轻量、柔和、可降级**。

### 10.1 动效时长

- 微反馈(按压、悬停):**0.15s**
- 标准过渡(淡入淡出、展开):**0.25s**
- 中等转场(抽屉、浮层):**0.3s**
- 大转场(页面切换):**0.4s**
- 最长不超过 **0.5s**

### 10.2 缓动曲线

- 标准:`cubic-bezier(0.4, 0, 0.2, 1)`(Material standard easing)
- 进入:`cubic-bezier(0.16, 1, 0.3, 1)`(弹性感,用于浮层出现)
- 退出:`cubic-bezier(0.4, 0, 1, 1)`(快速消失)
- **禁止线性动画**(`linear`)——除了加载旋转

### 10.3 核心动效模式

**按钮按压**
- 按下:`scale(0.97)`,0.1s
- 释放:回弹 `scale(1)`,0.15s

**卡片进入**
- 初始:`opacity: 0; transform: translateY(8px);`
- 结束:`opacity: 1; transform: translateY(0);`
- 列表项按顺序延迟 30ms(stagger)
- 0.3s 总时长

**输入框聚焦**
- 背景色从 `bg-grouped` 过渡到 `bg-card`,0.2s
- 外发光环从 0 扩展到 3px,0.2s

**Todo 勾选**
- 图标切换 + 0.2s 缩放弹性(1.0 → 1.2 → 1.0)
- 如果是卡片级 Todo:0.3s 后卡片淡化 + 向下滑入已完成分组

**重要标记**
- 书签图标从上方"落下"挂到卡片上,0.3s 带轻微摆动
- 取消时"飞走"淡出,0.2s

**Agent 浮窗展开**
- 从右下角缩放 + 淡入,0.35s cubic-bezier(0.16, 1, 0.3, 1)

**页面切换**
- 新页面从右侧滑入,旧页面向左淡出 + 小幅位移
- 0.35s

### 10.4 降级规则

必须支持 `prefers-reduced-motion: reduce` 媒体查询:

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

用户开启"减少动态效果"时,所有动画退化为**瞬间切换**,但保持最终视觉状态不变。

## 11. 多 Park 的视觉区分

每个 Park 应该有**细微但能感知**的视觉 identity,帮用户快速判断"我在哪里"。区分方式要**克制**——通过顶部区块、图标,而不是整体换色。

### 11.1 心记 Park

- 顶部大标题:"心记"
- 标题旁配 SF Symbol `heart.text.square.fill`
- 心情颜色(可选):当天主要心情的柔和色作为日历日期的标记色

### 11.2 做记 Park

- 顶部大标题:项目名
- 标题旁配 SF Symbol `square.stack.3d.up.fill`
- 顶部展示愿景与当前聚焦(目标区域)
- 记录流用类型颜色(见 §2.4)做左侧色条

### 11.3 知识库 Park

- 标题旁配 SF Symbol `books.vertical.fill`
- 左上角抽屉图标呼出仓库列表(F026 已定)
- 笔记卡片样式偏"阅读"感——行高稍宽松

### 11.4 计划 Park / 项目实验库

第一版未开发,占位即可。

### 11.5 全局 Todo / 重要视图

## 12. Do's 与 Don'ts

### ✅ Do

- 用系统字体栈(`-apple-system, SF Pro, PingFang SC`)
- 所有字号都用负字符间距(包括正文)
- Accent 只用琥珀金,重要/置顶/类型色各自独立
- 卡片圆角 14px(比 iOS 标准稍大更柔)
- 阴影多层叠加、柔和、向下散射
- 导航栏和抽屉用毛玻璃
- 用 SF Symbols 作所有 UI 图标
- 动效缓和、可降级、不超过 0.5s
- 正文左对齐,只有空状态和居中卡片可居中
- 所有交互有物理反馈(按压缩放、淡入、弹性)
- 按钮最小触摸目标 44px
- 支持浅色/深色模式

### ❌ Don't

- 不用 apple.com 营销页的黑白交替 cinematic 布局
- 不用 980px 胶囊 CTA 作为主按钮(分段控制器除外)
- 不用纯黑 `#000000` 做深色背景——用 `#1C1C1E`
- 不引入第二种 accent 色(除类型色外)
- 不用硬边框或实线分割大片区域
- 不用 emoji 作 UI 图标
- 不用宽字符间距(SF Pro 天生要紧)
- 不用字重 700+(除极少数 Large Title)
- 不用 inner shadow 或重阴影
- 不做不透明的导航栏(毛玻璃是 identity)
- 不做复杂的粒子/震动/爆炸式动画
- 不在正文段落里居中对齐
- 不用不带灰度的饱和色(纯红、纯绿、霓虹色)
- 不做"系统 App 假装"(不模仿 iOS 提醒事项、天气等 App 的完整界面)

### 13.5 迭代原则

1. 每个新 UI 元素先检查是否符合"柔软日记本"气质——克制、温暖、触感好
2. 颜色预算:只有琥珀金一种 accent,类型色只在做记用,其他一律中性色
3. 字体优先 iOS 系统栈,所有字号用负字符间距
4. 阴影多层柔和,大多数元素没有阴影
5. 毛玻璃用于导航、抽屉、浮窗这三类——其他地方不要随便用
6. 所有图标从 SF Symbols 取,不用 emoji
7. 所有动效缓和且可降级,尊重 `prefers-reduced-motion`
8. 所有按钮触摸目标 ≥ 44px
9. 不模仿 apple.com 的营销页风格,对标的是 Apple Journal / Notes / Reminders
